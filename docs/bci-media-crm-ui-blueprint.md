# BCI 自媒体矩阵获客系统 UI 蓝图

## 1. 左侧主菜单

推荐菜单顺序：

1. 工作台
2. 今日发布
3. 内容资产库
4. 真实资料库
5. AI 内容库
6. IP 矩阵
7. 账号矩阵
8. 招生 CRM
9. 数据复盘
10. 团队与权限

## 2. 角色首页

### 2.1 超级管理员首页

首页卡片：

- 今日待审核内容
- 今日待发布内容
- 本周新增线索
- 本周到访预约
- 各平台线索排行
- 各 IP 转化排行
- 团队待办异常

快捷入口：

- 新增账号
- 新增 IP
- 新增资料
- 分配账号负责人
- 导入 Notion 数据

### 2.2 部门负责人首页

首页卡片：

- 待审核内容
- 被驳回后重新提交内容
- 今日已发布
- 未回填数据内容
- 本周内容表现 Top 10
- 本周线索来源 Top 10

核心操作：

- 审核通过
- 驳回修改
- 查看账号发布完成率
- 查看 IP 内容表现

### 2.3 运营人员首页

首页卡片：

- 我负责的账号
- 今日待发布
- 审核通过待发布
- 被驳回待修改
- 待回填数据
- 本周发布进度

核心操作：

- 新建内容
- 提交审核
- 上传发布链接和截图
- 回填 3 日 / 7 日 / 30 日数据
- 写复盘

### 2.4 AI 内容编辑首页

首页卡片：

- 今日待生成内容
- 最近使用资料
- 好用 prompt
- 已采用 AI 稿
- 待人工修改稿

核心操作：

- 选择资料生成内容
- 选择 IP 改写语气
- 生成小红书 / 视频号 / 公众号版本
- 转为内容资产

### 2.5 招生顾问首页

首页卡片：

- 今日待跟进
- 新线索
- 已咨询
- 已预约到访
- 到访后待跟进
- 本月报名结果

核心操作：

- 新增跟进记录
- 更新线索阶段
- 查看线索来源内容
- 反馈线索质量

## 3. 关键页面

### 3.1 今日发布

用途：保证每天所有账号发出的内容都上传系统。

列表字段：

- 发布日期
- 平台
- 账号
- IP
- 内容标题
- 状态
- 负责人
- 发布链接
- 数据回填状态
- 是否产生线索

操作：

- 上传发布内容
- 上传截图
- 填写链接
- 回填数据
- 关联线索

### 3.2 内容资产库

用途：保存所有内容作品。

筛选：

- 年份
- 月份
- 平台
- 账号
- IP
- 主题
- 目标年级
- 内容状态
- 是否产生线索

详情页模块：

- 最终文案
- 口播稿
- 图片视频
- 调用资料
- AI 生成记录
- 审核记录
- 发布记录
- 数据表现
- 关联线索

### 3.3 真实资料库

用途：保存 BCI 的事实资料，可以被内容和 AI 调用。

筛选：

- WACE
- ATAR
- 大学录取
- 新加坡签证
- 学费
- BCI 资料
- 毕业生案例
- 竞品学校
- 家长 FAQ

详情页模块：

- 资料正文
- 图片 / 附件
- 来源
- 核实状态
- 可调用范围
- 被哪些内容调用
- 带来多少线索

### 3.4 AI 内容库

用途：保存 AI 生成过程和 prompt。

页面结构：

- 左侧：资料选择、IP 选择、平台选择、目标人群
- 中间：prompt 和生成结果
- 右侧：历史版本、采用情况、转内容资产按钮

关键动作：

- 保存 prompt
- 生成新版本
- 标记好用 / 一般 / 不采用
- 转为内容资产

列表字段：

- IP 类型
- 目标人群

详情页模块：

- 语言风格
- 主讲主题
- 禁止话题
- 常用话术
- 绑定账号
- 历史内容
- 线索表现

### 3.6 账号矩阵

用途：管理所有自媒体账号。

列表字段：

- 平台
- 账号名称

- 负责人
- 账号阶段
- 发布频率
- 粉丝数
- 本月发布数
- 本月线索数

详情页模块：

- Top 内容

### 3.7 招生 CRM

用途：管理内容带来的学生和家长线索。

看板阶段：

- 新线索
- 已联系
- 已咨询
- 预约到访
- 已到访
- 测试面试
- 录取
- 缴费
- 流失

线索详情页：

- 学生和家长信息
- 年级和课程意向
- 来源平台
- 来源账号

- 来源内容
- 跟进记录
- 下次跟进时间
- 报名或流失结果

### 3.8 数据复盘

用途：团队每周、每月、每季度复盘。

看板模块：

- 平台表现
- 账号表现
- IP 表现
- 内容主题表现
- 真实资料调用表现
- AI prompt 表现
- 招生漏斗
- 团队执行情况

关键指标：

- 发布数
- 浏览 / 曝光
- 收藏
- 私信
- 线索
- 到访
- 报名
- 线索转化率
- 到访转化率
- 报名转化率

## 4. 每日标准工作流

```text
运营 / AI 编辑创建内容
→ 调用真实资料和图片素材
→ 选择 IP 与账号
→ 提交部门负责人审核
→ 负责人通过或驳回
→ 运营发布到平台
→ 回系统上传发布链接、截图、正文和素材
→ 3 日 / 7 日 / 30 日回填数据
→ 有线索则进入 CRM
→ 周会复盘内容、账号和 IP 表现
```

## 5. MVP 首屏建议

第一版登录后不要做复杂首页，直接做成工作台。

运营人员首屏：

- 左边：我负责的账号
- 中间：今日待发布和审核状态
- 右边：待回填数据和最近线索

部门负责人首屏：

- 左边：待审核队列
- 中间：今日所有账号发布进度
- 右边：本周内容和线索表现

招生顾问首屏：

- 左边：今日待跟进
- 中间：线索阶段看板
- 右边：来源内容和线索质量
