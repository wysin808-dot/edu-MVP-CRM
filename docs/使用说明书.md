## 1. 系统概述

BCI 自媒体矩阵获客系统是一个面向国际学校（BCI WACE 7-12）招生团队的内容运营中台。系统将**内容生产 → 审核发布 → 数据回填 → 线索跟进**串联为完整闭环，覆盖小红书、视频号、公众号、抖音等多平台。

**核心理念：Content to Enrollment（内容到招生）。**

系统包含 11 个功能模块，通过左侧导航栏切换。不同角色登录后看到的导航菜单和数据范围不同。

## 2. 角色与权限

系统内置 5 个角色，通过页面右上角的「当前角色」下拉框切换。

| 角色 | 职责 | 可见模块 |
|------|------|---------|
| **运营人员** | 账号内容生产、提交审核、发布归档、数据回填 | 工作台、今日发布、内容库、资料库、内容日历、招生 CRM |
| **部门负责人** | 审核内容、监控账号发布进度、复盘内容效果 | 全部模块（除权限管理） |
| **超级管理员** | 管理用户、角色、账号、IP、资料库和全局数据权限 | 全部模块 |
| **AI 内容编辑** | 基于真实资料库生成内容，保存 Prompt 和版本记录 | 工作台、内容库、资料库、AI 内容、内容日历 |
| **招生顾问** | 跟进分配线索，记录到访、报名和流失结果 | 工作台、招生 CRM |

### 数据权限规则

- **运营人员 / AI 编辑**：今日任务和账号矩阵只显示自己负责的账号，内容库为全公司共享。
- **部门负责人 / 管理员**：查看全部账号和全部内容。
- **招生顾问**：仅查看分配给自己的线索，不显示内容模块。

## 3. 工作台

工作台是登录后的首页，提供当日工作全景。

### 3.1 KPI 卡片

页面顶部显示 3-4 个动态指标卡片，数据实时从内容管道和 CRM 计算：

| 卡片 | 含义 |
|------|------|
| **今日任务** | 当天分配给你的发布任务总数（运营人员只看自己账号的任务） |
| **待审核** | 当前处于"待审核"状态的内容数量 |
| **待回填数据** | 已发布但尚未补充阅读/点赞/线索等数据的内容数量 |
| **本周新线索** | CRM 中本周新增的招生线索数量 |

> 卡片数值会随角色切换自动更新。

### 3.2 内容管道

以状态色块展示所有内容的分布情况：

- **草稿**（蓝色）— 尚未提交审核的内容
- **待审核**（橙色）— 已提交，等待部门负责人审核
- **已驳回**（红色）— 审核未通过，需要修改后重新提交
- **可发布**（绿色）— 审核通过，可以发布
- **已发布**（灰色）— 已完成发布和归档

**操作：** 点击任意色块，弹窗显示该状态下的内容列表，可逐条查看或操作。

### 3.3 发布查询

按日期、平台、账号筛选已发布的内容记录，查看关联数据表现。

**操作步骤：**
1. 选择日期（默认今天）
2. 选择平台和账号（可选"全部"）
3. 点击「查询」
4. 点击表格中的行可查看详细数据或进行数据回填

### 3.4 快捷操作按钮

- **上传发布**：发布归档（填写发布链接、截图、正文）
- **创建计划**：新建一条内容计划

## 4. 今日发布

点击左侧「今日发布」进入。

### 4.1 我的今日任务

按时间排列显示当天的发布任务卡片，每张卡片包含：

- 发布时间（如 09:30）
- 平台标签（小红书 / 视频号 / 公众号）
- 任务状态标签（待发布 / 待审核 / 草稿修改 / 待归档）
- 内容标题
- 所属账号和 IP

**卡片右侧的操作按钮根据状态自动变化：**

| 任务状态 | 运营人员看到的按钮 | 部门负责人看到的按钮 |
|---------|------------------|-------------------|
| 待发布 | 详情 + 发布归档 | 详情 + 发布归档 |
| 待审核 | 详情（只读等待） | 详情 + 审核 |
| 草稿 / 草稿修改 | 详情 + 提交审核 | 详情 + 审核 + 提交审核 |
| 待归档 | 详情 + 发布归档 | 详情 + 发布归档 |

### 4.2 今日完成度

右侧面板显示当天的整体进度：计划发布数、已归档数、待审核数、待回填数据数。

## 5. 内容资产库

公司全部内容的统一管理库，全角色共享。

### 5.1 内容卡片

每条内容以卡片形式展示，包含标题、状态标签、账号、作者、发布日期等信息。

### 5.2 策略筛选器

内容库顶部提供三维策略筛选，帮助做"内容体检"：

- **漏斗阶段**：Awareness → Consideration → Trust → Visit → Enroll
- **情绪钩子**：反常识 / 焦虑共鸣 / 痛点 / 数字震撼 / 案例代入
- **复用状态**：可二改 / 可转（视频号/公众号）/ 已复用

左侧还有**主题簇**筛选：WACE / NUS·NTU / 学费 / 签证 / 毕业生案例。

### 5.3 内容详情与操作

点击任意内容卡片，弹窗显示详情。**不同角色和状态下的操作不同：**

#### 运营人员点击内容：

| 内容状态 | 弹窗内容 | 可执行操作 |
|---------|---------|---------|
| 草稿 | 内容详情（只读） | 「提交审核」按钮 |
| 待审核 | 内容详情（只读） | 无按钮（等待审核） |
| **已驳回** | **可编辑表单**：显示驳回原因，可修改标题、备注、CTA、内容类型、情绪钩子 | **「保存修改并重新提交」**按钮 |
| 可发布 / 审核通过 | 内容详情 | 「发布归档」按钮 |
| 已发布 / 待回填 | 数据回填表单 | 「保存数据」按钮 |

#### 部门负责人点击他人内容：

| 内容状态 | 弹窗内容 | 可执行操作 |
|---------|---------|---------|
| 待审核 / 草稿 / 已驳回 | 审核表单（含审核记录） | 通过 / 驳回 / 修改意见 |
| 已发布 / 待回填 | 数据查看 | 查看指标 |

> 部门负责人点击自己创建的内容时，按运营人员逻辑处理（因为负责人也运营账号）。

### 5.4 搜索

顶部搜索框支持按标题、账号、受众、CTA、关键词、主题簇进行全文搜索。

## 6. 真实资料库

存放所有经过事实核查的参考资料，是 AI 生成内容和人工写作的基础。

### 6.1 资料卡片信息

每条资料包含：

- **标题 / Claim**：资料核心事实
- **来源**：数据出处
- **学科 / Subject**：所属领域
- **引用次数**：被内容引用的次数
- **复查周期**：下次需要核实的时间
- **可见性**：公开 / 内部

### 6.2 操作

- **新增资料**：点击右上角「新增资料」
- **搜索**：按事实资料、来源、Subject、Type、Notes 搜索
- 点击卡片查看详情

## 7. AI 内容库

AI 辅助内容生成模块，包含调用面板和 Prompt 模板库。

### 7.1 调用设置

1. 选择**资料主题**（从真实资料库中选取）
2. 选择**目标 IP**（升学顾问 / 校长 / 招生老师）
3. 选择**目标平台**（小红书 / 视频号 / 公众号）
4. 点击「生成 3 个版本」

系统会调用 AI 生成适合该平台的多个版本，显示在右侧结果区。

### 7.2 Prompt 模板库

保存可复用的 Prompt 模板，每个模板包含：

- 模板名称和作者
- 适用阶段（标题 / 正文 / 改写）
- 目标平台和目标 IP
- Prompt 原文和输出示例
- 质量评分（1-5 星）
- 使用次数统计

> Prompt 被使用超过 2 次建议入库保存。

## 12. 数据复盘

部门负责人和管理员的数据分析模块。

### 12.1 平台线索表现

横向柱状图对比各平台（小红书 / 视频号 / 公众号 / 抖音）的线索获取量。

### 12.2 招生漏斗

五级漏斗可视化：曝光 → 私信 → 有效线索 → 到访 → 报名。

### 12.3 内容策略体检

四个维度的分析面板：

- **漏斗阶段分布**：内容是否覆盖了全漏斗？避免"头重脚轻"
- **情绪钩子分布**：哪种情绪钩子在用？避免单一打法
- **复用效率**：多少内容只在单平台发了？
- **WACE 内容追踪**：本周 WACE Focus 内容是否达标（≥ 2 条）

### 12.4 内容表现排行

按综合分排序的内容排行表，字段包括：阅读数、点赞、评论、私信、线索、综合分。

### 12.5 关键词表现追踪

追踪每个关键词覆盖的内容数、总阅读、总线索和覆盖漏斗阶段。

### 12.6 内容 A/B 测试

对比同一主题不同版本的表现差异，找出最优内容策略。

## 14. 核心工作流程

### 14.1 内容从创建到发布的完整流程

```
创建计划（草稿）
    ↓
运营人员编辑内容
    ↓
提交审核 → 状态变为「待审核」
    ↓
部门负责人审核
    ↓
  ┌─ 通过 → 状态变为「可发布」→ 运营人员用手机发布 → 发布归档
  │
  ├─ 驳回 → 状态变为「已驳回」→ 运营人员修改内容 → 重新提交
  │
  └─ 修改意见 → 记录意见 → 运营人员修改后重新提交
```

### 14.2 运营人员每日工作流程

1. **登录系统**，默认进入工作台
2. **查看 KPI 卡片**，了解今日任务量
3. **点击「今日发布」**，查看分配给自己的任务
4. 对于**待发布**的内容：用手机端发布后，回系统点「发布归档」
5. 对于**草稿**的内容：完善后点「提交审核」
6. 对于**已驳回**的内容：查看驳回原因 → 直接在弹窗中修改内容 → 点「保存修改并重新提交」
7. 对于**已发布**的内容：3 日/7 日/30 日后回填阅读、点赞、私信等数据

### 14.3 部门负责人每日工作流程

1. **登录系统**，查看工作台
2. **关注「待审核」数量**，点击色块查看待审列表
3. 逐条点击**「审核」**，查看内容详情和审核记录
4. 做出审核决定：**通过 / 驳回 / 修改意见**
5. 查看**数据复盘**，分析内容策略是否合理
6. 如果自己也运营账号，同时执行运营人员的工作流程

### 14.4 已驳回内容的修改流程

1. 运营人员在工作台或内容库看到「已驳回」标记
2. 点击后弹出**编辑表单**，顶部红色区域显示驳回原因
3. 直接修改以下字段：
   - 标题
   - 备注 / 正文要点
   - CTA 引导语
   - 内容类型（下拉选择）
   - 情绪钩子（下拉选择）
4. 填写**修改说明**（描述本次改了什么）
5. 点击**「保存修改并重新提交」**
6. 系统自动将状态改为「待审核」，并在审核记录中记录修改了哪些字段

### 14.5 发布归档流程

1. 内容状态为「可发布」或「审核通过」后，运营人员用手机在对应平台发布
2. 回到系统，点击「发布归档」或「上传发布」
3. 填写发布链接、截图、发布正文等信息
4. 提交后内容状态变为「已发布」
5. 等待 3 日 / 7 日 / 30 日后进行数据回填

### 14.6 数据回填流程

1. 点击已发布的内容，弹出数据回填表单
2. 填写实际数据：阅读数、点赞数、评论数、分享数、私信数、线索数
3. 点击「保存数据」
4. 数据自动关联到内容表现排行和平台线索分析

## 15. 常见问题

### Q: 为什么我看不到某些导航菜单？
**A:** 不同角色可见的模块不同。运营人员看不到 AI 内容、IP 矩阵、账号矩阵和数据复盘。切换到有权限的角色即可看到。

### Q: 为什么今日任务显示 0 条？
**A:** 运营人员只看到分配给自己账号的任务。如果当天没有你负责的账号的发布任务，就会显示 0。

### Q: 内容被驳回了怎么办？
**A:** 点击已驳回的内容，系统会弹出编辑表单，顶部显示驳回原因。直接修改内容后点「保存修改并重新提交」即可。

### Q: 部门负责人可以自己发内容吗？
**A:** 可以。负责人点击自己创建的内容时，系统自动切换为运营人员视角，可以编辑和提交。点击他人内容时则进入审核模式。

### Q: 内容库是每个人各自的还是公司共享的？
**A:** 内容库是**全公司共享**的，所有角色都能看到全部内容。但今日任务和账号矩阵是按个人账号分配筛选的。

### Q: 如何追踪一条线索的来源？
**A:** 在招生 CRM 中，每条线索卡片都标注了来源内容和平台。在数据复盘中可以看到各平台的线索表现对比。

### Q: 通知在哪里？
